# PRD 模板

## Frontmatter

```yaml
---
版本: "{版本号}"
标题: "{标题}"
日期: "{YYYY-MM-DD}"
状态: 草稿 | 评审中 | 已确认
前置: "{依赖的 PRD 版本号，如 PRD 1.0}"
---
```

## 文档结构

### 1. Scope (范围)

> 一句话说明本 PRD 要解决什么问题、交付什么功能。

### 2. 背景 & 动机

- 当前痛点或不足
- 为什么现在做（数据支撑、用户反馈等）

### 3. 目标

用可量化的指标描述成功标准，例如：
- 用户留存提升 X%
- 响应时间降低到 Y ms
- 转化率达到 Z%

### 4. 方案设计

#### 4.1 功能 / 架构

- 核心逻辑描述
- 参数定义用表格：

| 参数 | 含义 | 默认值 | 范围 |
|------|------|--------|------|
| `param_name` | 说明 | 默认 | [min, max] |

#### 4.2 数据流

```
输入 → 处理步骤 → 输出
```

#### 4.3 接口变更

- 新增 / 修改的 API endpoint
- 新增 / 修改的前端组件
- 数据库 schema 变更

### 5. 实施计划

| 阶段 | 内容 | 涉及文件 |
|------|------|----------|
| Phase 1 | ... | `path/to/file` |
| Phase 2 | ... | ... |

### 6. 风险 & 回退

- 已知风险及缓解措施
- 回退方案

### 7. 验证计划

- 单元测试覆盖点
- 集成测试场景
- 用户验收标准

### 8. Review 记录

| 日期 | 审阅人 | 结论 | 备注 |
|------|--------|------|------|
| YYYY-MM-DD | ... | 通过/修改 | ... |
