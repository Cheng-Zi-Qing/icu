## 1. Scope (范围)

> v1.0 实现基于有限状态机（FSM）的健康状态管理核心功能，包括 4 种工作状态切换、动态水合算法、健康提醒记录与每日/每周报告生成。

**产品定位：** 一款专为极客与重度脑力工作者设计的轻量级、非侵入式健康状态管理桌宠。

**核心理念：** 摒弃死板的定时器，采用事件驱动（手动切换状态）与有限状态机（FSM）架构。通过无缝切入开发者的工作流（心流），实现”0 打扰”的医学级健康干预。

---

## 2. 背景 & 动机

### 当前痛点

- 传统定时提醒工具打断心流，降低工作效率
- 固定间隔的健康提醒不考虑个体差异（体重、杯容量）
- 深度专注时被强制打断，引发用户反感
- 缺少健康行为的长期追踪与反馈

### 为什么现在做

- 远程办公普及，久坐与屏幕时间显著增加
- 开发者群体对"非侵入式"工具有强烈需求
- macOS 桌宠生态成熟（PySide6 + rumps 技术栈稳定）
- 本地 AI（Ollama）可实现个性化文案，无隐私风险

## 4. 方案设计

### 4.1 科学理论依据

本产品的提醒逻辑与底层算法严格遵循以下人体工学与视觉健康研究标准：

- **数字眼疲劳 (DES) 与 20-20-20 法则：** 每直视屏幕 20 分钟，需看向 20 英尺（约 6 米）外的物体 20 秒，以缓解睫状肌痉挛。
    
- **久坐微干预 (Active Micro-breaks)：** 连续坐姿超过 45-60 分钟会引起腰背肌肉温度异常与僵硬。需每 45 分钟进行 2-3 分钟的主动物理拉伸。
    
- **认知水合作用：** 轻度脱水（1-2%）会导致逻辑推断能力下降。最佳方案为在工作期间进行高频、小口径的补水，避免单次过量增加肾脏负担。

系统的核心是一个由用户完全手动控制的状态机。系统启动时默认为资源零消耗的「待机」状态。

#### 1. 🛌 待机/下班状态 (Idle / Off-Work)

- **触发：** App 启动默认，或手动选择“下班”。
    
- **逻辑：** 彻底销毁所有后台健康计时器，不占用系统运算资源。
    
- **UI表现：** 桌宠缩成屏幕角落的灰色小图标，或呈现盖被子睡觉的动画。系统释出当日健康与饮水结算报告。
    

#### 2. 💻 工作/活跃状态 (Working)

- **触发：** Menu Bar 菜单选择”开始工作”。
    
- **逻辑：** 启动主循环，护眼 (20分钟)、拉伸 (45分钟)、补水 (动态间隔) 并行倒计时。
    
- **UI表现：** 保持清醒，眨眼，或根据系统键盘敲击频率呈现敲击小键盘的动画。
    

#### 3. 🔕 深度专注/会议状态 (Focus / Meeting)

- **触发：** 轮盘菜单选择“进入专注”。
    
- **逻辑：** 冻结（挂起）所有健康计时器，屏蔽一切弹窗。启动“专注时长”正向秒表，记录健康负债。
    
- **UI表现：** 戴上降噪耳机或打坐，变为半透明状态并退至屏幕绝对边缘。
    

#### 4. ☕ 暂离/餐饮状态 (Break / Away)

- **触发：** 轮盘菜单选择“暂离”。
    
- **逻辑：** 彻底重置并暂停所有健康计时器（离开屏幕期间不产生健康负债）。
    
- **UI表现：** 呈现干饭、喝咖啡或发呆动画。再次选择“回来工作”时，直接切回工作状态并重新计时。

### 4.3 动态水合算法

摒弃固定时间的喝水提醒，系统将在初始化时获取用户的“体重”与“常用杯子容量 (V_cup)”，并动态重载提醒频率。

**算法参数：**

| 参数 | 含义 | 默认值 | 范围 |
|------|------|--------|------|
| `body_weight` | 用户体重 (kg) | 70 | [40, 150] |
| `cup_volume` | 杯子容量 (ml) | 300 | [200, 800] |
| `work_hours` | 预计工作时长 (h) | 8 | [4, 12] |
| `hydration_ratio` | 工作期间需水占比 | 0.65 | [0.5, 0.8] |

**1. 目标设定算法：**

- 每日总需水量 = 体重(kg) × 35ml
    
- 工作期间需水量（核心 KPI） = 每日总需水量 × 65%
    

**2. 动态触发频率：**

- 每日需喝杯数 = 工作期间需水量 ÷ 杯子容量 (向上取整)
    
- 动态提醒间隔 = 设定的工作总时长 ÷ 每日需喝杯数
    
- _安全阈值约束：计算出的间隔若 < 30分钟，提示更换大杯子；若 > 120分钟，系统强制修正为 90分钟极限值。_
    

**3. 数据交互与闭环：**

- UI 融入实体水滴进度条（如：显示 4 个半透明水杯图标代表今日目标）。
    
- 提醒弹出时，提供多梯度反馈按钮：[一口干了 (+100%)] / [喝了半杯 (+50%)] / [稍后再喝]。

### 4.4 健康提醒与行为记录

~~**专注 ≥ 60 分钟：** 桌宠呈现”虚脱”状态，强制触发最高级别补偿机制（2 分钟锁屏拉伸 + 补水）。支持极客密码（如输入 `sudo break`）强行跳过以应对线上紧急故障。~~ ← 2026-03-15 改为非强制提醒+记录模式

#### 提醒机制（非强制）

当用户从”🔕 深度专注”退出，切回”💻 工作状态”时，系统进行负债结算并**弹出提醒**（不锁屏）：

- **专注 < 30 分钟：** 无缝恢复此前冻结的计时器
- **30 ≤ 专注 < 60 分钟：** 弹出轻量级提醒：”专注了 X 分钟，建议喝水 + 远眺 20 秒”
- **专注 ≥ 60 分钟：** 桌宠呈现”虚脱”状态，弹出高优先级提醒：”专注了 X 分钟，强烈建议拉伸 2-3 分钟 + 补水”

#### 用户反馈按钮

提醒弹窗提供 3 个选项：
- **[已完成]** - 记录为”响应并执行”，计入健康行为达成率
- **[稍后]** - 记录为”延迟响应”，10 分钟后再次提醒
- **[忽略]** - 记录为”未响应”，不再提醒，计入每日报告

#### 数据记录字段

每次提醒记录到 SQLite：

| 字段 | 类型 | 说明 |
|------|------|------|
| `timestamp` | DATETIME | 提醒时间 |
| `reminder_type` | TEXT | 护眼/拉伸/补水 |
| `focus_duration` | INT | 专注时长 (秒) |
| `user_response` | TEXT | completed/delayed/ignored |
| `response_time` | INT | 响应延迟 (秒) |

### 4.5 每日/每周报告

#### 每日下班报告 (Daily Report)

切换至”下班状态”时，桌宠背起书包离开，弹出当日小票：

**报告内容：**

1. **工作时长统计**
   - 总工作时长：X 小时 Y 分钟
   - 深度专注次数：N 次，累计 Z 分钟
   - 暂离次数：M 次

2. **健康行为达成率**
   - 护眼提醒：响应 A / 总计 B 次（达成率 X%）
   - 拉伸提醒：响应 C / 总计 D 次（达成率 Y%）
   - 补水目标：已喝 E ml / 目标 F ml（达成率 Z%）

3. **家庭任务派发**
   - 回家需补水：G ml（= 每日总需水量 - 公司已喝水量）
   - 温馨提示：睡前 2 小时避免大量饮水

#### 每周综合报告 (Weekly Report)

每周日晚或下周一早，生成周报（可通过 Menu Bar 手动查看）：

**报告内容：**

1. **本周工作概览**
   - 总工作时长：X 小时
   - 日均工作时长：Y 小时
   - 深度专注总时长：Z 小时（占比 P%）

2. **健康行为趋势**
   - 护眼提醒平均响应率：X%（折线图）
   - 拉伸提醒平均响应率：Y%（折线图）
   - 每日饮水达成率：Z%（柱状图）

3. **改进建议**
   - 基于响应率 < 50% 的提醒类型，给出个性化建议
   - 例如："本周拉伸响应率仅 35%，建议调整拉伸间隔或设置更明显的提醒"

#### 动画状态映射

**动画实现方案：静态图 + 代码动画引擎**

用户只需提供 **1 张静态形象图**（PNG，透明背景），App 通过代码动画引擎生成所有状态动画。

**内置动画效果：**

| 状态 | 动画效果 | 实现方式 |
|------|----------|----------|
| `idle` | 上下浮动 + 眨眼 | `y_offset = sin(t*2)*5`，每 3 秒闭眼 0.1 秒 |
| `working` | 左右摇摆 | `x_offset = sin(t*3)*3` |
| `focus` | 缩小 + 半透明 | `scale = 0.8, opacity = 0.5` |
| `break` | 旋转 + 放大 | `rotation = sin(t)*10°, scale = 1.1` |
| `stretch` | Y 轴拉伸 | `scale_y = 1 + sin(t*4)*0.2` |
| `exhausted` | 倾斜 + 抖动 | `rotation = -15°, x_offset = random(-2,2)` |
| `happy` | 跳跃 + 放大 | `y_offset = abs(sin(t*5))*20, scale = 1.2` |

### 4.7 数据持久化 Schema

**SQLite 表结构：**

```sql
-- 状态切换记录
CREATE TABLE state_transitions (
    id INTEGER PRIMARY KEY,
    timestamp DATETIME,
    from_state TEXT,
    to_state TEXT,
    duration_seconds INT
);

-- 健康提醒记录
CREATE TABLE health_reminders (
    id INTEGER PRIMARY KEY,
    timestamp DATETIME,
    reminder_type TEXT,  -- eye/stretch/water
    focus_duration INT,
    user_response TEXT,  -- completed/delayed/ignored
    response_time INT
);

-- 饮水记录
CREATE TABLE water_intake (
    id INTEGER PRIMARY KEY,
    timestamp DATETIME,
    volume_ml INT,
    cup_percentage INT  -- 100/50
);

-- 每日汇总
CREATE TABLE daily_summary (
    date DATE PRIMARY KEY,
    work_duration INT,
    focus_duration INT,
    eye_response_rate REAL,
    stretch_response_rate REAL,
    water_completion_rate REAL
);
```

为确保在 macOS 环境下的极致轻量与高可控性，系统底层依赖以下技术方案：

| 技术 | 用途 | 备注 |
|------|------|------|
| `PySide6` | 核心 UI | 支持 macOS 无边框透明窗口与边缘吸附 |
| `rumps` | Menu Bar 入口 | 构建极简的顶部菜单栏状态切换 |
| `transitions` | 状态引擎 | 实现稳定的四种状态流转 |
| `SQLite` | 数据持久化 | 轻量级本地存储，生成报告 |
| `Ollama API` | AI 文案生成（可选） | 本地大模型生成个性化提醒文案 |

#### AI 扩展说明（v1.0 可选）

- **调用时机：** 生成每日/每周报告时，调用本地 Ollama API（如 `qwen3.5:27b`）
- **输入数据：** 当周健康行为统计数据（响应率、达成率）
- **输出内容：** 个性化改进建议（傲娇/毒舌风格可选）
- **降级方案：** API 调用失败时，使用预设模板文案

## 6. 风险 & 回退

### 已知风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 用户忘记切换状态 | 计时器数据不准确 | 增加"智能检测"：系统休眠/锁屏时自动暂停计时 |
| macOS 权限限制 | 无法获取键盘敲击频率 | 降级为静态动画，不依赖系统事件 |
| PySide6 性能问题 | 桌宠动画卡顿 | 优化动画帧率，使用 GPU 加速 |
| 用户不愿手动切状态 | 产品使用率低 | 提供"自动模式"：基于屏幕活动自动切换工作/暂离 |

### 回退方案

- **Phase 1-3 失败：** 降级为纯 Menu Bar 工具（无桌宠 UI）
- **AI 集成失败：** 使用预设文案模板
- **SQLite 损坏：** 自动备份机制，每日导出 CSV
