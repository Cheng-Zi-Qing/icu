# Model Config And Avatar Studio Design

**Date:** 2026-03-27

**Status:** Approved in browser review

## Goal

把现有“生成配置”从混合式长表单拆成两类页面：

- `配置页面` 只负责三类模型能力配置
- `更换形象页面` 负责主题风格、桌宠形象动画、话术三类内容生成与预览应用

## Product Split

### 1. 配置页面

配置页面只保留模型能力配置，不承载创作、生成、预览、应用流程。

能力类型固定为：

- 文本描述
- 动画形象
- 代码生成

每个能力页都使用相同的信息结构：

- 基础配置
  - 服务商
  - 模型
  - 接口地址
- 高级设置
  - 认证 JSON
  - 选项 JSON

高级设置默认折叠。

配置页面的视觉结构使用顶部 tabs，而不是左侧导航。

### 2. 更换形象页面

更换形象页面是统一创作台，负责三类生成内容：

- 主题风格
- 桌宠形象动画
- 话术

页面使用顶部 tabs，单主面板结构，视觉风格完全对齐当前 `PixelTheme`。

## Interaction Model

三个创作 tab 统一使用同一套交互闭环：

1. 输入 `prompt`
2. 生成预览
3. 不满意则重新生成
4. 满意后应用

这里的关键词必须是 `prompt`，不使用 `vibe`。

## Theme Style Tab

`主题风格` tab 负责主题生成，不直接暴露模型编辑表单。

页面内容应包含：

- 当前已应用主题摘要
- 本次生成会使用的模型摘要
  - 文本描述模型
  - 主题代码模型
- 跳转到相应模型配置页的入口
- prompt 输入区
- 预览区
  - 必须包含桌宠气泡 / 状态 chip 预览
  - 必须覆盖右键菜单与配置表单这类 GUI 组件
- 生成预览 / 重新生成 / 应用 操作区

生成得到的结果先进入 `draft`，只有点击“应用”才覆盖当前 `applied` 主题。

## Avatar Animation Tab

`桌宠形象动画` tab 负责生成桌宠图像与动作预览。

模型摘要来自：

- 文本描述
- 动画形象

结果同样分为：

- `draft`
- `applied`

## Speech Tab

`话术` tab 负责生成角色话术草稿与示例预览。

预览区必须包含真实的桌宠气泡弹出预览，而不是只展示纯文本列表。

结果同样分为：

- `draft`
- `applied`

## Visual Direction

浏览器草图已确认按当前 `PixelTheme` 对齐，不引入新画风。

必须复用当前主题气质：

- 深绿背景
- 浅绿强调色
- Menlo / monospace 像素感排版
- 低饱和分层面板
- 轻边框、高对比文本、像素风按钮与输入框

## Non-Goals

本次不做：

- 新主题画风探索
- 模型能力类型重命名或重构
- Auth / Options 的结构化键值编辑器
- 真正的多主题运行时切换系统扩展

## Implementation Notes

- `GenerationConfigWindowController` 退回成纯模型设置页
- 主题生成逻辑从配置页移出，迁到更换形象工作台
- 更换形象工作台应承接“生成 -> 预览 -> 再生成 -> 应用”的交互闭环
